# Architecture

## 当前结构

`game/`
作用：核心规则、数据模型、战斗和楼层逻辑。

`cli/`
作用：命令行入口、渲染和输入处理。

`content/`
作用：怪物等内容配置。

`tests/`
作用：核心逻辑回归测试。

`ai_dev/`
作用：AI 开发规则、路线图、任务、会话记录。

## 当前核心状态

`Player`
持有玩家属性、经验、等级、金币、成长逻辑。

`Monster`
持有怪物属性和奖励信息。

`GameState`
持有楼层、玩家、战斗日志、是否结束。

## 当前主循环

1. 进入楼层
2. 生成怪物
3. 玩家确认战斗
4. 执行战斗
5. 结算经验和金币
6. 升级
7. 进入下一层或死亡结束

## 当前约束

1. 先保持同步回合制战斗
2. 先不实现背包和存档
3. 随机生成必须可控且简单
4. 怪物内容从 JSON 读取
