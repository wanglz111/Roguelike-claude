# Architecture

## 当前结构

`game/`
作用：核心规则、数据模型、战斗、楼层和事件逻辑。

`cli/`
作用：命令行入口、渲染和输入处理。

`content/`
作用：怪物、物品、事件等内容配置。

`tests/`
作用：核心逻辑回归测试。

`ai_dev/`
作用：AI 开发规则、路线图、任务、会话记录。

## 当前核心状态

`Player`
持有玩家属性、经验、等级、金币、背包、装备槽（武器、护甲）、成长逻辑、事件效果处理。

`Monster`
持有怪物属性、奖励信息、掉落物、Boss 标记。

`Item`
持有物品名称、类型、效果、描述、装备槽、属性加成。

`Event`
持有事件名称、描述、选项列表、最低楼层。

`EventChoice`
持有选项文本、效果类型、效果值、结果文本。

`GameState`
持有楼层、玩家、战斗日志、是否结束。

`SaveLoad`
提供存档与读档功能，将游戏状态保存为 JSON 文件。

## 当前主循环

1. 进入楼层
2. 生成怪物（每 5 层生成 Boss）
3. 玩家确认战斗
4. 执行战斗（使用装备加成后的属性，支持暴击，Boss 战有特殊提示）
5. 结算经验和金币
6. 掉落物品（药水或装备）
7. 随机事件（50%概率触发）
8. 玩家可使用背包物品、装备装备或保存游戏
9. 升级
10. 进入下一层或死亡结束

## 战斗系统

1. 回合制战斗，玩家先攻击，怪物后攻击
2. 伤害计算：max(1, 攻击力 - 防御力)
3. 暴击系统：10% 概率触发，造成 2 倍伤害
4. 玩家和怪物都可以触发暴击
5. 暴击在战斗日志中有特殊提示

## 当前约束

1. 先保持同步回合制战斗
2. 装备系统已实现，事件系统已实现，存档系统已实现，Boss 系统已实现
3. 随机生成必须可控且简单
4. 怪物、物品和事件内容从 JSON 读取
5. 所有文本内容支持中英文双语
6. 事件有 50% 概率在战斗后触发
7. 存档保存在用户主目录的 .roguelike_saves 文件夹中
8. Boss 每 5 层出现一次（第 5、10、15 层等），拥有更高的属性和奖励
