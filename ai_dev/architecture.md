# Architecture

## 当前结构

`game/`
作用：核心规则、数据模型、战斗和楼层逻辑。

`cli/`
作用：命令行入口、渲染和输入处理。

`content/`
作用：怪物、物品等内容配置。

`tests/`
作用：核心逻辑回归测试。

`ai_dev/`
作用：AI 开发规则、路线图、任务、会话记录。

## 当前核心状态

`Player`
持有玩家属性、经验、等级、金币、背包、装备槽（武器、护甲）、成长逻辑。

`Monster`
持有怪物属性、奖励信息、掉落物。

`Item`
持有物品名称、类型、效果、描述、装备槽、属性加成。

`GameState`
持有楼层、玩家、战斗日志、是否结束。

## 当前主循环

1. 进入楼层
2. 生成怪物
3. 玩家确认战斗
4. 执行战斗（使用装备加成后的属性）
5. 结算经验和金币
6. 掉落物品（药水或装备）
7. 玩家可使用背包物品或装备装备
8. 升级
9. 进入下一层或死亡结束

## 当前约束

1. 先保持同步回合制战斗
2. 装备系统已实现，存档暂未实现
3. 随机生成必须可控且简单
4. 怪物和物品内容从 JSON 读取
5. 所有文本内容支持中英文双语
