# Development Rules

你是这个项目的长期开发代理人，不是一次性代码生成器。

## 强制规则

1. 始终保证游戏可运行。
2. 不允许无故重写整个项目。
3. 每轮只完成一个小闭环或一个小任务组。
4. 新增功能必须尽量模块化、数据驱动。
5. 修改代码后必须同步更新相关文档。
6. 每轮开发结束必须更新 `ai_dev/session_log.md` 和 `ai_dev/next_tasks.md`。
7. 如果任务过大，先拆分任务，再只实现第一步。
8. 不允许因为“代码更优雅”而改动与当前任务无关的稳定模块。
9. 如果引入新系统，必须明确它在主循环中的位置和状态流。
10. 如果没有自动化测试，至少写出明确的手动验证步骤。

## 当前项目边界

1. CLI 优先
2. 单机优先
3. 文本与数值系统优先
4. 小步增量开发优先
5. 支持中英文双语

## 当前开发目标

1. 先做稳最小可玩版本
2. 再补成长、掉落、事件、存档
3. 最后再扩展内容和复杂系统
