# Changelog

### Combat Log UI Improvements (战斗日志 UI 改进)

**视觉优化**
1. Boss 战添加醒目的视觉分隔符（═ 字符）
2. Boss 战显示完整属性信息（HP、攻击、防御）
3. 战斗结果添加视觉分隔符（─ 字符）
4. 胜利/失败添加图标标识（✓ 胜利 / ✗ 失败）
5. 所有战斗添加 ⚔️ 图标

**技术实现**
1. 更新 `game/combat.py` 的 fight() 函数日志格式
2. 保持向后兼容，不改变游戏逻辑
3. 仅影响显示层，测试无需修改

### New Game+ System (周目系统)

**周目系统实现**
1. 通关后提示玩家是否开始新周目
2. 新周目保留：玩家等级、装备（武器、护甲、饰品）、一半金币
3. 新周目重置：楼层回到 1、HP/MP 恢复满、清空战斗日志
4. 怪物难度缩放：根据周目数应用倍率（cycle 2: 1.3x, cycle 3: 1.6x, cycle 4: 1.9x）
5. 存档系统支持周目数保存和加载
6. 修复存档系统缺失的 MP 和职业相关字段

**技术实现**
1. GameState 新增 cycle 字段追踪当前周目
2. generate_monster 函数支持周目参数，应用难度倍率
3. 主循环集成新周目提示和状态重置逻辑
4. 存档系统完整支持所有玩家属性

### Milestone 4: 扩展系统完成

**技能系统 (Skill System)**
1. 新增 MP 系统：玩家拥有 MP 属性，升级时增加
2. 实现 6 种主动技能：强力打击、治疗术、防御、狂暴、轻度治疗、坚守
3. 技能消耗 MP，在战斗前选择使用
4. 支持三种效果类型：伤害倍率、治疗、防御提升
5. 提供高低成本选项，增加战术深度

**职业系统 (Class System)**
1. 实现 3 种职业：战士、法师、盗贼
2. 不同职业有不同基础属性（HP、MP、攻击、防御）
3. 不同职业有不同成长速率（每级属性增长不同）
4. 游戏开始时选择职业，影响整个游戏体验

**内容扩展**
1. 怪物池扩展到 19 种（15 普通 + 4 Boss），覆盖 1-20 层
2. 事件池扩展到 14 个，包含永久属性提升事件
3. 新增 4 种事件效果类型：boost_attack、boost_defense、boost_max_hp、trade_boost_attack

**Bug 修复**
1. 修复随机种子硬编码问题，确保每次游戏体验不同
2. 修复治疗技能未使用 total_max_hp 的问题
3. 修复事件效果逻辑错误

**测试与平衡**
1. 新增职业系统测试（6 个测试）
2. 更新平衡测试包含 MP 系统验证
3. 所有 21 个测试通过

## 2026-03-12

### 核心系统实现

**背包与装备系统 (Inventory & Equipment)**
1. 实现物品系统：药水、武器、护甲
2. 实现背包系统：收集、使用、装备物品
3. 装备提供属性加成（攻击、防御、HP）
4. 战斗系统使用装备后的总属性

**战斗增强 (Combat Enhancement)**
1. 实现暴击系统：10% 概率触发，造成 2 倍伤害
2. 玩家和怪物都可以触发暴击
3. 暴击在战斗日志中有特殊提示

**事件系统 (Event System)**
1. 实现随机事件系统：50% 概率触发
2. 初始 4 个事件，后扩展到 10 个
3. 支持多种效果类型：治疗、伤害、金币、交易等

**商店系统 (Shop System)**
1. 实现 3 种商店类型：流浪商人、铁匠铺、大师工匠
2. 商店生成概率：普通楼层 30%，Boss 层后 50%
3. 支持库存管理和购买验证

**存档系统 (Save/Load System)**
1. 实现完整的存档与读档功能
2. 存档保存在用户主目录的 .roguelike_saves 文件夹
3. 支持保存完整游戏状态：玩家、背包、装备、楼层

**Boss 系统 (Boss System)**
1. 实现 Boss 层机制：每 5 层出现 Boss
2. Boss 拥有更高的属性和奖励
3. Boss 战有特殊提示和战斗标记

**内容扩展**
1. 怪物池从 4 个扩展到 15 个（11 普通 + 4 Boss）
2. 覆盖 1-20 层的完整游戏体验

**数值平衡**
1. 实现平衡测试框架
2. 测试并调整 Boss 属性
3. 验证早期生存和属性成长
